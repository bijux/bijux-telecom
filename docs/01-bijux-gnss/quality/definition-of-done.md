# Definition Of Done

A command documentation or code change is done when the public intent is
complete, reviewable, and honestly proven.

## Done Means

- the owning command family is explicit
- the handbook still describes the changed boundary or contract correctly
- the relevant local proof has been run or the gap is explicitly called out
- public command meaning has not changed silently
- the commit boundary matches one durable command intent
