# Definition Of Done

A nav documentation or code change is done when the scientific intent is
complete, reviewable, and honestly proven.

## Done Means

- the owning scientific family is explicit
- the handbook still describes the changed boundary or contract correctly
- the relevant local proof has been run or the gap is explicitly called out
- public scientific meaning has not changed silently
- the commit boundary matches one durable scientific intent
